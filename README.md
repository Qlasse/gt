# GymTracker — Telegram Mini-App для трекинга рабочих весов

Минималистичный трекер весов: штанга визуализируется блинами, тяга блока — стопкой
с пином, гантель — растущими дисками. Плавный степпер и скраб-жест, избранное,
история и график прогресса по каждому упражнению.

**Память переживает любой перезапуск бота**: все данные хранятся в Telegram
CloudStorage (на серверах Telegram, привязаны к аккаунту пользователя). Бот —
только «кнопка входа», состояния у него нет.

## Структура

```
webapp/   — статичный mini-app (HTML/CSS/JS, без сборки)
bot/      — минимальный бот на grammY, отдаёт кнопку запуска
```

## Запуск за 4 шага

### 1. Создайте бота

В Telegram напишите [@BotFather](https://t.me/BotFather) → `/newbot` → задайте имя
и username. Сохраните полученный **токен**.

### 2. Разместите webapp на HTTPS-хостинге

Mini-app — обычная статика, подойдёт любой бесплатный хостинг:

- **GitHub Pages**: залейте репозиторий, в Settings → Pages включите раздачу.
  URL будет вида `https://<user>.github.io/<repo>/webapp/`.
- **Cloudflare Pages / Netlify / Vercel**: укажите папку `webapp` как корень.

Обязателен HTTPS — Telegram не откроет mini-app по HTTP.

### 3. Запустите бота

```bash
cd bot
npm install
copy .env.example .env    # Linux/macOS: cp .env.example .env
# впишите в .env свой BOT_TOKEN и WEBAPP_URL
npm start
```

### 4. Подключите кнопку меню (опционально)

В @BotFather: `/mybots` → ваш бот → **Bot Settings → Menu Button** → укажите тот же
URL. Тогда приложение будет открываться и с кнопки меню в чате, без `/start`.

## Локальная разработка

Откройте `webapp/index.html` через любой статик-сервер, например:

```bash
npx http-server webapp -p 5173
```

Вне Telegram приложение автоматически использует localStorage вместо
CloudStorage, так что весь функционал можно проверять прямо в браузере.
Чтобы проверить в самом Telegram без деплоя, пробросьте локальный сервер наружу
(например, `cloudflared tunnel --url http://localhost:5173`) и укажите
полученный HTTPS-адрес в BotFather.

## Данные

| Ключ CloudStorage | Содержимое |
|---|---|
| `exercises` | список упражнений: имя, тип снаряда, шаг, вес грифа, избранное, текущий вес |
| `log:<id>`  | история записей упражнения: дата, вес, подходы×повторы (до 60 записей) |
